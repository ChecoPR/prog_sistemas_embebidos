# Entrada y Salida (Conceptos básicos)

## Flujos de Entrada/Salida (I/O Streams)

```cpp
#include <iostream>

int main () {
  int i;
  // leer un valor en i
  std::cin >> i;               
  // imprimir el valor de i
  std::cout << i << '\n'; 
}
````

---

## Fuentes y destinos de datos

### Flujos de E/S de la terminal

* `std::cin`
  Caracteres desde **stdin**
  Lee desde un *buffer*.

* `std::cout`
  Caracteres hacia **stdout**
  Escribe primero en un *buffer*,
  se envía a la consola cuando el buffer se llena.

* `std::clog`
  Caracteres hacia **stderr**
  Escribe primero en un *buffer*,
  se envía a la consola cuando el buffer se llena.

* `std::cerr`
  Caracteres hacia **stderr**
  Escribe **inmediatamente** en la consola (sin buffer).

---

## Operadores de flujo (Stream Operators)

* `>>`  → “obtener de”
  `fuente >> destino`

* `<<`  → “enviar a”
  `destino << fuente`

Funcionan con:

* Tipos fundamentales
* Cadenas (`string`)

(Se puede agregar soporte para otros tipos.)

### Comportamiento de `>>`

* Lee hasta encontrar el siguiente carácter de espacio en blanco:

  * espacio
  * tabulador
  * salto de línea
  * etc.

### Encadenamiento de operadores

```cpp
int i = 0;
double d = 0.0;

// leer 2 valores de distinto tipo
std::cin >> i >> d;

// imprimir 2 valores y texto
std::cout << "tu entrada:\n"
          << i << " y " << d << '\n';
```

## ¡Evita usar `std::endl`!

Es común ver algo como esto en ejemplos o tutoriales de C++:

```cpp
std::cout << "algún texto" << std::endl;
std::cout << "más texto" << std::endl;
```

### ¿Por qué evitarlo?

* Cada uso de `std::endl`:

  * Inserta un salto de línea
  * **Fuerza el vaciado del buffer de salida**

* Esto puede causar **una degradación seria del rendimiento** si se usa frecuentemente.

Los flujos de E/S en C++ usan *buffers* para reducir el costo de las operaciones de entrada y salida del sistema.
La salida se acumula hasta que se puede escribir una cantidad mínima de caracteres.

El uso excesivo de `std::endl` **interfiere con este mecanismo**.

---

## Haz esto en su lugar

```cpp
std::cout << "algún texto\nmás texto\n";
```

Ventajas:

* `\n` inserta un salto de línea
* No fuerza el vaciado prematuro del buffer
* Solo una llamada al operador `<<`
  (cada llamada adicional puede generar una pequeña sobrecarga)

---

## ¿Cuándo usar `std::endl`?

Usa `std::endl` **solo si realmente necesitas** asegurarte de que la salida aparezca inmediatamente en pantalla.
