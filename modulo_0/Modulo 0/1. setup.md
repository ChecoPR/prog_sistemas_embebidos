# C++ Development Setup

## Herramientas

**Consejo:** comienza con un editor de texto + un compilador por línea de comandos.

Pero una vez que entiendas cómo funciona el proceso de construcción (*build*) en C++ y cómo usar un compilador desde la línea de comandos, deberías considerar cambiar a un **IDE** para proyectos más grandes.

---

## Code Editors & IDEs

### [Visual Studio Code](https://code.visualstudio.com/)

Probablemente una de las mejores opciones para principiantes hoy en día.

- Interfaz gráfica agradable
- Ofrece funciones básicas de IDE:
  - Autocompletado
  - Ir a definiciones
- Existe un plugin de **VIM** altamente configurable

Editor con algunas características de IDE.

Soporte común para:

- Linux
- MacOS
- Windows + WSL

---

### [VIM](https://www.vim.org/)

- Editor muy potente (curva de aprendizaje alta)
- Puede usarse directamente dentro de la terminal

---

### [Qt Creator](https://www.qt.io/development#ide)

- IDE
- Bastante amigable para principiantes

---

### [Dev-C++](https://www.embarcadero.com/free-tools/dev-cpp)

- Amigable para principiantes
- Configuración rápida y sin complicaciones

---

### [Microsoft Visual Studio IDE](https://visualstudio.microsoft.com/es/)

- Puede resultar abrumador para principiantes

---

### [JetBrains CLion](https://www.jetbrains.com/clion/)

- Puede resultar abrumador para principiantes

## Windows

Si usas Windows 10/11, se recomienda ampliamente utilizar **Windows Subsystem for Linux (WSL)**, que básicamente te da una línea de comandos Linux completa.

La mayoría de los recursos de aprendizaje, especialmente en C++, asumen un entorno Linux.  
Si deseas dedicarte profesionalmente al desarrollo de software, de cualquier forma necesitarás familiarizarte con Linux.

### Recursos para WSL

- [Instrucciones de instalación de WSL](https://learn.microsoft.com/en-us/windows/wsl/install)
- [Configurar VS Code para C++ en WSL](https://code.visualstudio.com/docs/cpp/config-wsl)
- [Introducción a Windows Terminal](https://devblogs.microsoft.com/commandline/getting-started-with-windows-terminal/)

## Compilers

### gcc / g++

Compilador por defecto en GNU/Linux y muy maduro.

#### Ubuntu / Windows + WSL

```bash
sudo apt install -y g++
sudo apt install -y build-essential
````

#### Sin WSL

- [WinLibs Project (recomendado)](https://winlibs.com/)
- [STL MinGW distribution](https://nuwen.net/mingw.html)
- [Dev-C++ incluye gcc integrado](https://www.embarcadero.com/free-tools/dev-cpp) (probablemente la opción más fácil para principiantes)

---

### clang / clang++

#### Ubuntu / Windows 10/11 + WSL

```bash
sudo apt install -y clang++
sudo apt install -y build-essential
```

#### Windows sin WSL

- [Clang Prebuild Binaries for Windows](https://releases.llvm.org/download.html)

## Browser-Based Compilers

### [Compiler Explorer](https://godbolt.org/)

- Ejecuta código usando muchos compiladores diferentes
- Ideal para pruebas rápidas
- Gran variedad de compiladores y versiones (incluyendo características experimentales)
- Incluye varias bibliotecas populares
- Excelente editor de texto:

  - Cursores múltiples
  - Modo VIM
  - Modo oscuro
- Muestra código ensamblador anotado de forma clara

---

### [Wandbox](https://wandbox.org/)

- Ejecuta código con distintos compiladores
- Ideal para pruebas rápidas
- Ventana de entrada y salida por consola
